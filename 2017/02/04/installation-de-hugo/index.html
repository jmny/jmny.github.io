Comment creer votre blog avec Hugo

Published Sat, Feb 4, 2017 by Louis PERRIER

Qu’est ce que Hugo

C’est un générateur de site statique. dans la meme famille on retrouve Jekill. Les cas d’utilisations sont :

  • Une home de projet open source
  • Un blog
  • Une homepage de site public
  • Un portefolio
  • et tout ce que peut gérer un site html statique

Le site du projet est gohugo.io

Machine de travail

Je suis sous ubuntu 16.04, mais cet exemple d’installation fonctionne sur Linux en général et Mac OS X à priori, il faut juste installer snap avec la methode adhoc sour votre OS.

Installer Hugo

sudo apt install snapd # lire la doc si ça ne fonctionne pas chez vous
snap install hugo
hugo version

(Plus de details ici.)

Creer son blog

hugo new site monblog

# pour voir ce qui a été généré ( soyons curieux :) )
cd monblog
tree -a

(La documention est là.)

Il faut un peu de config, donc ça se passe dans … et oui dans config.toml ! Mettons y le minimum vital.

echo "baseurl = \"https://example.com/\"
title = \"Mon blog\" " > config.toml

Ecrire son premier article

hugo new post/protocols-war.md
echo "
+++
date = \"2017-03-04T23:39:13+01:00\"
title = \"Protocols war\"
draft = true
+++

on connait tous ça :) 

 \![ ](https://imgs.xkcd.com/comics/standards.png)

" > content/post/protocols-war.md

Je vous invite à modifier le post, avec peut être un contenu plus personnel. Via le cartouche du header, vous pouvez également renseigner des information comme l’etat brouillon, le titre du post, ou les tags associés. Notez que dans le header il faut remplir l’attribut title=“” sinon le titre sera vide.

(la documentation est par ici.)

Choisir un theme et l’installer

Avec hugo, pas de theme par defaut. Je vous laisse vous perdre dans la gallerie de thème disponible et choisir le votre (ou le faire. On peux même en tester plusieurs, alors lachons nous.

git clone https://github.com/jbub/ghostwriter themes/ghostwriter
git clone https://github.com/dim0627/hugo_theme_robust.git themes/robust

Faisons nous plaisir et regardons le resultat. Option -D pour générer aussi les brouillons, par default hugo serve surveille les modifications et regénérer automatiquement, plus d’info avec : hugo help serve. Options -t pour indiquer le theme, mais ça vous aviez devinez. Ici on test donc le theme “robust”. Vous trouverez sur la page du thème des infos utiles comme des customisations possibles à ajouté via le fichier config.toml.

hugo serve -D -t robust

ou

hugo serve -D -t ghostwriter

Et voila

Vous pouvez maintenant visualiser votre blog, sur http://localhost:1313/. La prochaine fois on le publie sur gitpage.
